指标说明

  • Seed：随机数种子。每个 seed 会生成一张不同的 AllToAllV sendCounts 矩阵，但每个源 rank 的总发送量都固定为 16MiB。
  • Dst CV：目的 rank 收到的数据总量的不均衡系数，计算方式是 std(dst_total_bytes) / mean(dst_total_bytes)。值越大，说明流量越集中到少数目的 rank；0 表示每个目的 rank 收到的数据完全一样。
  • Baseline / Matrix / ClosV3：对应算法的全局吞吐，单位 GB/s，按全部 task 数据量除以全局 makespan 计算。
  • Matrix vs base 和 ClosV3 vs base：相对同一个 seed 下 baseline 的全局吞吐变化。负数表示比 baseline 慢。
  • 链不均衡：最重依赖链的数据量 / 平均依赖链的数据量。这里的“链”就是同一个逻辑并行单元上串行执行的一组 task。值越大，说明某个 slot/plane 被分到更多数据，更容易形成拖尾。

结论

  • 随机性确实存在，但不是偶发现象：matrix/closv3 对 seed 很敏感，核心相关指标是 chain_bytes_max_over_mean，也就是最重 slot 链相对平均链的倍数。
  • 最差 matrix seed=20260622，global=1111.53 GB/s，最慢链：0->4:4.45MiB 0->8:1.88MiB 0->12:0.59MiB。
  • 最差 closv3 seed=20260621，global=1043.77 GB/s，最慢链：0->13:1.95MiB 0->10:1.60MiB 0->7:4.55MiB。
  • baseline 的多 TP/full 模型对随机 V 更鲁棒；matrix/closv3 如果固定 peer->slot/plane，需要按 size 做负载均衡或拆大流。
