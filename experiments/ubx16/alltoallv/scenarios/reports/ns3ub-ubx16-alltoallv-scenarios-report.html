核心结论

  • 当 V 分布接近均匀，Matrix / MeshClos V3 可以把跨组流量铺到 4 个 clos 平面，global 吞吐从 3631.29 GB/s 提到 3844.14 GB/s，提升 5.9%。
  • 跨组占比很高但仍均匀时，MeshClos V3 仍有收益：从 3062.52 GB/s 到 3174.03 GB/s，提升 3.6%。
  • 一旦 AllToAllV 出现目的热点或源热点，当前 Matrix / MeshClos V3 的“peer 绑定单平面/单 slot”会让热点大流拖尾；轻微随机下 V3 比 baseline 低 54.3%，dispatch 热目的下低 13.5%。
  • 这个结果说明优化算法对均匀 AllToAll 很有效，但对 MoE 类 AllToAllV 需要进一步做大 peer 流拆分，或者按 size 重新做平面选择，否则热点 V 会吞掉分平面收益。

定量原因

  • Baseline 的 transport_channel 行数更多，模型里一个大的 peer 流可以被多 TP 分摊；所以在 dispatch/combine 热点分布下，baseline 虽然调度粗糙，但热点流不一定被单条 50GB/s 平面卡死。
  • MeshClos V3 / Matrix 当前 strict 建模更接近“每个 peer 选择一个 clos 逻辑平面”。均匀时这是优势，因为 15 个 peer 自然分散；热点时这是劣势，因为最大的 peer 数据量达到 4.00MiB，单 peer 拖尾决定 makespan。
  • Rank0 数值不能单独代表全局，尤其 combine 场景 hot source 是 rank 3/7/11/15；报告同时给 global GB/s 和热点 rank profiling。
