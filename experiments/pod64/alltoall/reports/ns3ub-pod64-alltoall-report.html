POD64 AllToAll 仿真报告

拓扑为 64 rank，每 8 rank 一组；每卡 8 个 clos 平面端口和 7 个组内 mesh 直连端口。覆盖 16/64/128/256/512/1024MiB/rank，所有大包 case 使用 MTP 并关闭 port/packet trace，仅保留 task trace 用于 profiling。

最佳全局吞吐27665.39 GB/s
最佳算法/数据量MeshClos V3 strict step-barrier 1024MiB
V3 step 1024MiB 单卡432.75 GB/s
V3 step 理论达成率96.1% / 96.2%

结论

  • V3 strict step-barrier 是这组实验里最接近网络上限的口径：1024MiB/rank 全局 27665.39 GB/s，rank0 432.75 GB/s。
  • 按 POD64 均匀 AllToAll direct payload 理论上限估算，全局上限约 28800 GB/s，单卡上限约 450 GB/s；V3 step 1024MiB 达成 96.1% / 96.2%。
  • baseline phase-barrier 1024MiB/rank 为 23806.67 GB/s，V3 step 相对它提升 16.2%。
  • V3 step 从 256MiB 到 1024MiB 仍有小幅增长：27203.62 → 27463.07 → 27665.39 GB/s，说明大包下趋于平台但 512MiB 还未完全饱和。

建模口径

  • baseline：HCCL Mesh1D，concurrent=16，full-TP；phase-barrier 是同步对照。
  • matrix strict：8x8 matrix 调度，组内 slot 走 mesh，跨组 slot 走固定 clos 平面；round-barrier 是同步对照。
  • MeshClos V3 strict：group-size=8，8 个 clos 平面；组内任务走直连 mesh，跨组任务按 V3 shift/link 映射到固定平面。
  • MeshClos V3 strict step-barrier：诊断对照。Mesh2D 任务保持并发，只强制每个 MeshClos 逻辑 step 全局完成后再进入下一 step。
  • 完整数据见 ns3ub-pod64-alltoall-summary.csv；每个 profile 链接展示 rank0 的 task 级开始/结束时间、数据量和带宽。
