POD64 AllToAll 性能对比

带宽口径已统一为：单rank带宽 = 单rank发送总数据量 / 全局最后完成时间。主表比较 baseline 16线程、baseline 64线程、MeshClos 固定平面 group-round sendrecv 三组。

结论

固定平面 vs baseline 16线程
+13.0%
固定平面 vs baseline 64线程
+6.1%
baseline 64线程 vs baseline 16线程
+6.5%

MeshClos 固定平面 group-round sendrecv 的单rank带宽为 418.85 GB/s，高于 baseline 16线程的 370.55 GB/s，也高于 baseline 64线程的 394.75 GB/s。

全局带宽 = 全部任务发送总数据量 / 全局最后完成时间。单rank带宽只用一个 rank 的发送总数据量做分子，但分母仍取全局最后完成时间。

  • group-round sendrecv 从 16MiB 到 128MiB 明显上升：309.34 -> 347.83 -> 418.85GB/s，小数据量下同步/启动开销占比更高。
  • 128MiB 到 512MiB 基本进入平台：418.85、412.08、429.46GB/s，和 stepbarrier 参考扫描的 420.65、425.06、429.11GB/s 接近。
  • 1024MiB group-round 按全局完成时间口径掉到 354.25GB/s，是异常长尾点；全局最慢 rank 是 51/28，完成时间约 3031us。
  • 因此当前趋势判断应看 16-512MiB：固定平面 group-round sendrecv 随数据量增大上升并进入约 420-430GB/s 平台；1024MiB 需要单独排查长尾，不应解释为算法大包退化。

数据量扫描参数

  • rank 数：64；group size：8；每组 8 rank，共 8 组。
  • 扫描数据量：16、64、128、256、512、1024 MiB/rank。
  • baseline 16线程：RTP 包喷洒，16 个通信线程。
  • baseline 64线程：RTP 包喷洒，每个远端 peer 一个逻辑线程，实际 63 个远端 peer 线程。
  • MeshClos 固定平面：group-round sendrecv，同一 group-round 内双向 sendrecv 同步，不做整轮强 barrier。
